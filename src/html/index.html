    Description

    This tool analyzes chat logs and looks for entosis reports with dedicated format.
    It then shows a synthetic view of on-going entosis.

    By default, the tool parses fleet chat (it needs to be started after the fleet creation or it will load last fleet logs !).
    You can also use another file by clicking the "+" button.

    Commands

    It recognize the following format : [system_name] / [node_id] / [status] / [time_left]
    Example : DO6H-Q / Z99 / friendly / 0:55

    [status] can be one of the following values :

    • new - node just spawned,
    • friendly - an ally is entosissing,
    • enemy - an enemy is entosissing,
    • neutral - nobody is entosissing,
    • unknown - you don't know if someone is entosissing,
    • vanished - the node disapeared.

    Note that "new" and "neutral" are basically the same thing and are shown as "neutral". You can use either command as you prefer (or both).

    [time_left] must follow a strict format : [minutes]:[seconds]. You can also use the warmup keyword instead of a timer.

    Tips and tricks

    You can click on a system name to show nodes details.

    The tool selects the last fleet file by default. If you have several characters logged-in in different fleets, make sure to double-check as Murphy law says that you will never get the right one !

    I highly recommand you to drag and drop system name instead of writting it manually.
    Even if the application tries to infer your input in case of typo, it is not really reliable. Drag and drop will be faster than wrtting full name anyway.

    Commands are not case sensitive.
